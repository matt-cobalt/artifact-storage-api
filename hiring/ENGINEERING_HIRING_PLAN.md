## Interview Process

### Standard Process (All Roles)

**Step 1: Application Review (1-2 days)**
- Resume review
- Portfolio/code samples review (if applicable)
- Initial screening (fit, experience, location)

**Step 2: Phone Screen (30 minutes)**
- Basic qualifications check
- Experience discussion
- Role fit assessment
- Salary expectations

**Step 3: Technical Assessment (1-2 hours)**
- **Backend/Frontend/Full-Stack:** Coding challenge (take-home, 2-4 hours)
  - Problem: Build a small feature (e.g., API endpoint, React component)
  - Submission: GitHub repo or code file
  - Evaluation: Code quality, problem-solving, best practices
- **DevOps:** Infrastructure challenge (take-home, 2-4 hours)
  - Problem: Set up CI/CD pipeline, deploy application
  - Submission: GitHub repo with documentation
  - Evaluation: Infrastructure knowledge, automation, security

**Step 4: Technical Interview (60 minutes)**
- Review technical assessment (discuss solution, alternatives)
- System design discussion (architecture, scaling)
- Coding exercise (pair programming, live coding)
- Technical depth (relevant technologies, best practices)

**Step 5: Culture Fit Interview (30 minutes)**
- Founder interview (Matt)
- Company culture, values, mission
- Role expectations, growth opportunities
- Questions from candidate

**Step 6: Reference Checks (1-2 days)**
- 2-3 professional references
- Previous managers, colleagues
- Verification: Experience, performance, culture fit

**Step 7: Offer (1 day)**
- Written offer (email)
- Compensation package (salary, equity, benefits)
- Start date (flexible, typically 2-4 weeks)

**Total Timeline:** 1-2 weeks from application to offer

---

### Interview Process (Engineering Lead)

**Additional Step:** Leadership Interview (60 minutes)
- Experience leading teams (examples, challenges)
- Technical decision-making (architecture, technology choices)
- Conflict resolution, mentoring
- Scaling teams (1 → 10 engineers)

**Panel Interview:** 
- Founder (Matt)
- 2-3 engineers (if team exists)
- Questions: Technical leadership, team management, culture

## Onboarding Plan

### 30-Day Onboarding Schedule

**Week 1: Foundation**

**Day 1: Welcome & Setup**
- Welcome meeting (founder + team)
- Company overview (mission, vision, culture)
- Tool setup (GitHub, Slack, Railway, Supabase access)
- Codebase access (GitHub repos, documentation)
- Accounts created (all necessary services)

**Day 2-3: Codebase Deep Dive**
- Review technical architecture documentation
- Explore codebase (key files, structure, patterns)
- Review existing features (dashboards, APIs, agents)
- Q&A sessions with founder/team

**Day 4-5: First Task**
- Small bug fix or feature (non-critical)
- Code review process (submit PR, get feedback)
- Git workflow practice (branches, commits, PRs)
- Deploy to staging (test deployment process)

**Week 1 Goal:** Understand codebase, complete first PR

---

**Week 2: Product Understanding**

**Day 6-7: Product Deep Dive**
- Product roadmap review
- Customer use cases (real examples)
- Demo: Complete customer journey
- Feature walkthrough (all major features)

**Day 8-10: Integration & Testing**
- Integration with team (standups, code reviews)
- Testing workflows (unit tests, integration tests)
- Debugging process (logs, monitoring, troubleshooting)
- Q&A sessions (technical questions)

**Week 2 Goal:** Understand product, integrated with team

---

**Week 3: First Real Feature**

**Day 11-14: Feature Development**
- Assigned first real feature (P1 priority)
- Feature specification review
- Implementation (coding, testing)
- Code review process (submit PR, address feedback)
- Deploy to staging (test, verify)

**Week 3 Goal:** Complete first feature, deploy to staging

---

**Week 4: Production Deployment**

**Day 15-17: Production Prep**
- Feature testing (comprehensive testing)
- Documentation (update docs if needed)
- Production deployment (deploy, monitor)
- Customer validation (verify feature works for customers)

**Day 18-20: Reflection & Planning**
- Onboarding review (what worked, what didn't)
- Feedback session (founder + engineer)
- Next feature planning (roadmap priorities)
- Goal setting (30-60-90 day goals)

**Week 4 Goal:** First feature in production, ready for more

---

### Onboarding Success Metrics

**Week 1:**
- ✅ First PR merged
- ✅ Codebase explored (read key files)
- ✅ Tools set up (all accounts created)

**Week 2:**
- ✅ Product understood (can explain to others)
- ✅ Integrated with team (participates in standups)
- ✅ Testing workflows understood

**Week 3:**
- ✅ First feature complete (code written, tested)
- ✅ PR approved (code review passed)
- ✅ Deployed to staging

**Week 4:**
- ✅ First feature in production
- ✅ Customer feedback positive (feature works)
- ✅ Ready for next feature
